📜 Data Dictionary & Logic

TABLE: auth_user (Django Built-In)

The core identity table for all staff members (Warden, Medical Staff, Security Staff). Authentication is handled via Djoser & SimpleJWT. Clearances are managed via the standard groups ManyToMany relationship.

Field Type Constraints Purpose
id Integer Primary Key System unique identifier
username Varchar(150) Required, Unique Login credential and Audit Trail identifier
groups ManyToMany auth_group Determines role-based access (Medical, Security, etc)
is_active Boolean Default: True Used for soft-deleting staff without breaking Audit FKs

TABLE: arkham_app_cellblock

The facility infrastructure ledger. This table tracks the physical cell blocks and enforces capacity limits during inmate admissions.

Field Type Constraints Purpose
id Integer Primary Key System unique identifier
name Varchar(20) Required, Unique Name of the physical block (e.g., Block-A)
max_capacity Integer Default: 10 Admission Engine utilizes this to reject overflows

🔍 LOGIC: Features a dynamic current_count property that reverse-calculates active inmates on the fly.

TABLE: arkham_app_inmateprofile

The primary entity representing individuals incarcerated within the Asylum.

Field Type Constraints Purpose
id Integer Primary Key System unique identifier
name Varchar(100) Required Legal or birth name of the inmate
alias Varchar(50) Required Known criminal moniker or supervillain persona
cell_block_id Integer Required, FK Link to CellBlock table. Shielded by Transfer Safety Engine.
status Varchar(20) Default: ACTIVE ACTIVE or DISCHARGED (Soft Delete mechanism)

TABLE: arkham_app_medicalfile

Stores highly sensitive psycho-medical data. Access to this table is strictly throttled and restricted to Medical Staff & Super Admins.

Field Type Constraints Purpose
id Integer Primary Key System unique identifier
inmate_id Integer Unique, FK Link to InmateProfile (Strict 1:1)
assigned_to_id Integer FK (auth_user) The clinical owner (Doctor). Dictates read access.
referral_diagnosis Varchar(200) Required Initial reason for admission (Enforced by Admission Engine)
internal_diagnosis Text Blank=True Ongoing clinical assessment (Default: Pending evaluation)
meds Varchar(150) Required Current prescribed medication regimen

⚠️ RELATIONSHIP: One-To-One. Created atomically with InmateProfile using transaction.atomic().

TABLE: arkham_app_auditlog

A chronological, immutable record of all system activity. Read actively by the Transfer Safety Engine.

Field Type Constraints Purpose
id Integer Primary Key System unique identifier
user_id Integer FK (auth_user) The actor behind the request
actor_name Varchar(100) Required Username snapshot (preserves log if user is deleted)
actor_group Varchar(50) Required Role snapshot
action_type Varchar(50) Required CREATE, UPDATE, DELETE, or DETAILED_READ
target_model Varchar(50) Required e.g., InmateProfile, MedicalFile
target_id Integer Required The PK of the affected entity
timestamp DateTime Auto-set Exact server time of the event
